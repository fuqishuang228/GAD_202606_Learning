## 1. 实验目的

当前 raw10 no-context prototype-router baseline 的核心特征是：

```
nCN,
CP1, CP2, CP3, CP4,
d_nCN,
d_CP1, d_CP2, d_CP3, d_CP4
```

之前诊断显示：

```
High-order expert = CP3, CP4, d_CP3, d_CP4
```

是最稳定的信号，而 Local expert 很弱。

AtlasULP 的 local refinement 使用：

```
HN: Hierarchical Neighborhood Overlap
RA: Resource-Allocated Overlap
LC: Local Connectivity Density
```

所以这次请实现一个最小 Atlas-inspired 版本：

```
Local expert = 1-2 hop HN / RA / LC
High-order expert = 原始 CP3 / CP4 / d_CP3 / d_CP4
```

不要改 high-order CP 计算方式。

## 2. 新特征定义

对 query edge `(u, v)`，基于当前 rolling history graph `G_t`。

定义 frontier：

```
F_u^1 = u 的 1-hop frontier
F_u^2 = u 的 2-hop frontier

F_v^1 = v 的 1-hop frontier
F_v^2 = v 的 2-hop frontier
```

这里建议使用 **exact frontier**，也就是：

```
F_u^1 = N(u)
F_u^2 = 距离 u 恰好为 2 的节点集合
```

`F_u^2` 不应该包含 `u` 自己，也不应该包含 1-hop 节点。
`v` 同理。

对于：

```
a, b ∈ {1, 2}
```

计算以下三组特征。

### 2.1 HN

```
HN_a_b(u, v) = |F_u^a ∩ F_v^b|
```

也就是两个 frontier 的重叠节点数量。

### 2.2 RA

```
RA_a_b(u, v) = sum_{w in F_u^a ∩ F_v^b} 1 / max(deg(w), 1)
```

也就是对 overlap 节点按 degree 降权。

### 2.3 LC

```
LC_a_b(u, v) = number of edges between F_u^a and F_v^b
```

也就是 `F_u^a` 这个节点集合和 `F_v^b` 这个节点集合之间有多少条边。

如果用无向图，请注意不要重复计数。可以按现有 undirected adjacency 的语义实现，但最终每对节点间的边只算一次。

### 2.4 log1p 压缩

对所有 HN / RA / LC 特征做：

```
log1p(x) = log(1 + x)
```

也就是输出：

```
log1p(HN_a_b)
log1p(RA_a_b)
log1p(LC_a_b)
```

不要做 snapshot-relative z-score。

## 9. 对比 baseline

请和之前的 btc\_otc raw10 no-context proto-router baseline 对比：

```
raw10 final: 0.6968 AUROC / 0.1433 AUPRC
raw10 High-order expert: 0.6990 AUROC / 0.1438 AUPRC
```

这次重点看：

### 9.1 Local 是否变强

如果：

```
atlas_local_k2 Local expert 明显高于 raw10 Local expert
```

说明 Atlas local refinement 有用。

### 9.2 High-order 是否保持

如果：

```
atlas_local_k2 High-order expert ≈ raw10 High-order expert
```

说明我们没有破坏最强 CP3/CP4 信号。

如果 High-order 明显下降，需要优先检查 feature indexing / expert grouping 是否错位，因为 High-order 输入理论上没变。

### 9.3 Final 是否提升

如果：

```
atlas_local_k2 final > raw10 final
```

说明 Atlas-local K=2 不只是增强 Local，也真的帮到了最终 MoE。

如果：

```
Local 提升但 final 没提升
```

说明 local feature 有信息，但 router/fusion 没利用好。

如果：

```
Local 没提升，final 也没提升
```

说明 local refinement 这条线可能仍然不是跨域 DGAD 的主信号。
